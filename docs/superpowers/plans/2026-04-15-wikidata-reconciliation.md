# Wikidata Reconciliation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Reconcile button to the Wanted Categories view that queries the Wikidata reconciliation API, shows candidates per row, and lets the user link a Wikidata item when creating a category via "Create WI".

**Architecture:** The frontend calls the Wikidata reconciliation API directly (no backend proxy needed). When "Create WI" is clicked with a selected QID, the backend creates the Commons category page as before, then calls the Wikidata API to add a P373 claim and a commonswiki sitelink to the matched item, using the existing `merge_sdc_statements` logic to avoid overwriting existing data.

**Tech Stack:** Vue 3 + TypeScript (frontend), FastAPI + Python 3.13 (backend), Wikidata reconciliation API (`https://wikidata-reconciliation.wmcloud.org/en/api`), Wikidata MediaWiki API (`https://www.wikidata.org/w/api.php`)

## Task 4: `useReconcile` composable

**Files:**
- Create: `frontend/src/composables/useReconcile.ts`
- Create: `frontend/src/composables/__tests__/useReconcile.test.ts`

The composable calls `https://wikidata-reconciliation.wmcloud.org/en/api` directly from the browser using `fetch`. Titles are batched into one request (all visible rows at once). Underscores in titles are replaced with spaces for the query.

**Types** (define inside the composable file or in a co-located types file):
```typescript
interface ReconCandidate {
  id: string        // QID e.g. "Q123"
  name: string
  description: string
  score: number
  match: boolean
}
```

**State:**
- `reconcileResults: Ref<Record<string, ReconCandidate[]>>` — keyed by raw title (underscores)
- `selectedQids: Ref<Record<string, string>>` — title → selected QID
- `isReconciling: Ref<boolean>`

**Functions:**
- `reconcile(titles: string[])` — batch query all titles, auto-select when exactly one result, populate `reconcileResults`
- `toggleSelect(title: string, qid: string)` — select if unselected, deselect if already selected
- `clearReconcile()` — reset all state

- [ ] Write the failing test in `useReconcile.test.ts`:
